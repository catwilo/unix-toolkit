## context system
- MACRO (this file): global state — machines, all repos, pending tasks, do-NOT, last-known-good
- MICRO: ~/unix-toolkit-tools/<repo>/.ctx.md — per-repo stack, fixes, pending, last-known-good
- Both updated same turn as fix. Never deferred.
- OWNER: miko-task owns macro + all micro ctx. ut does not read or write ctx files.
- Session start: read macro first, then micro of repos being touched.

## ut — scope (finalized)
Commands: sync | status | push | run | list | clone | health | diff | tag | add | rm | machines
- machines: reads noemap cache → shows devices alive, OS detected, repo count, sync state
- status: only repos with something to report; summary line format:
    [32 repos] ✓ 28 synced  ⚡ 3 ahead  ✗ 1 conflict
    d0 [debian]:31  d1 [macos]:18  tx [android]:31
- ut push: primitive; miko-task calls it internally

## miko-task — scope (finalized)
- Task manager + owner of all ctx files (macro + all micro)
- Reads repos.tsv to know which repos have micro-ctx and where
- Does NOT write repos.tsv (ut owns it)
- DSTASK_DATA: ~/unix-toolkit-tasks/ (separate repo, tag: data,priv)

## miko sync — flow (finalized)
1. dstask sync          — sync tasks with remote
2. ut fetch             — fetch all remotes, no merge (non-destructive)
3. ut status            — show real cross-repo state
4. reconcile            — conflict resolver (see below)
5. commit               — miko sync -m "msg" (non-interactive) | miko sync (interactive prompt)
6. ut push              — push all verified repos

## reconcile — behavior (finalized)
- Syncs every repo it can without information loss
- Real conflict (same file, both sides): shows brief diff + 2-3 line hint, marks pending, continues
- Never blocks full flow for one conflicted repo
- Never discards local or remote without explicit confirmation
- Output: all repos end clean+pushed OR explicit list of what stayed pending and why
