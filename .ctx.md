Diseño propuesto:

**Umbral**: 900 KB por chunk (seguro bajo el límite ~1 MB de Android)
**Trigger**: `BYTES > PAGER_LIMIT` en cualquier backend
**Split**: `split -b 900k "$TMP"` → dir temporal, cleanup en trap
**Avance**: `read -n1 -s -r key < /dev/tty` — funciona aunque stdin esté piped
**Abort**: tecla `q`
**UI**:
```
[1/3] 900 KB — any key: next · q: abort
[3/3] done
```

**Cambios en clipso.sh**:
1. Añadir `PAGER_LIMIT=$((900*1024))` junto a `MAX_BYTES`
2. Extraer dispatch actual → función `do_copy()`
3. Añadir `paginate()` que itera chunks, reasigna `TMP` por chunk, llama `do_copy`
4. En dispatch final: `(( BYTES > PAGER_LIMIT )) && paginate || do_copy`

Minimal — sin tocar backends, sin romper flujo normal. ¿Confirmás para emitir el patch?
