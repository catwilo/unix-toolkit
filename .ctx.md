## clipboard Mac<-Termux — MECHANISM (how it works)
- clipso (Termux) writes to ~/.local/share/noemap/clip.sock when SSH session detected
- Mac runs nclip-listen (via launchd io.clipd.agent, KeepAlive) listening on ~/.noemap-clip.sock
- ssh -R maps Termux socket -> Mac socket; data flows Termux clipso -> Mac pbcopy
- zsh-setup --mac (Darwin-guarded) generates io.clipd.agent.plist + RemoteForward in ~/.ssh/config
- RECOVERY if pbcopy hangs: zsh-setup fix_clipboard  (bootout agent + pkill -9 nc/pbcopy + killall pboard)
- REQUIRES on Termux sshd: StreamLocalBindUnlink yes — DONE
